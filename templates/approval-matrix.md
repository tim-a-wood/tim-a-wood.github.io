# Approval Matrix

| Decision Type | Who Can Decide | Who Must Be Consulted | Escalation Required |
|---|---|---|---|
| Bug fix (no behavior change) | Founder | QA | No |
| UI/UX change (cosmetic) | Founder | QA | No |
| New feature (non-AI) | Founder | QA, Marketing | No |
| New AI feature | Founder | QA, Legal, Cybersecurity | Yes — pre-launch review |
| Pricing change | Founder | Finance, Marketing, Legal | Yes |
| Public marketing claim | Founder | Marketing, Legal | Yes if involves AI claims |
| Data collection change | Founder | Legal, Cybersecurity | Yes |
| New external API dependency | Founder | Cybersecurity, Legal | Yes |
| Open source dependency add | Founder | Legal (license check) | If GPL/AGPL |
| External partnership | Founder | Legal, Finance | Yes |
| Release / launch | Founder | QA, Legal, Cybersecurity, Marketing | Yes — launch readiness |
| Incident response (P0) | Founder | All relevant specialists | Yes — immediate |
| Spend > $500 | Founder | Finance | Yes |
| Any legal document | Founder + real attorney | Legal agent | Yes — always |

## Notes
- Agents may proceed autonomously only on items marked "No" in the Escalation Required column
- When in doubt, escalate — the cost of asking is lower than the cost of a mistake
- Legal agent never substitutes for a real attorney on material decisions
