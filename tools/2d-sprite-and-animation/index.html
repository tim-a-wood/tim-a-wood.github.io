Intake

Prompt-only entry still works, but structured art-direction fields now feed the concept prompt builder directly. Reference rows can point to local files or browser uploads and are copied into the project before generation.

ComfyUI is the default and only real concept backend. The debug backend exists only as an explicit fallback for local testing.

References

Typed references are stored under each project and passed into the concept backend through uploaded conditioning images. Identity, costume, style, and prop references stay separate in metadata.

Unsupported prompt patterns are rejected before concept generation. Existing projects load with backward-compatible defaults even if they predate the structured brief schema.

Concept Review

Initial runs generate six concepts. Refinement and regenerate-similar runs generate four. Review actions, lineage, and derived metrics all come from history.json rather than UI-only state.

Run Summaries

The most recent run is selected by default. Old projects with legacy concept boards still load, but they will not show the same lineage detail until new runs are created.

Compare

The approved concept stays fixed on the left when available. Click compare on any other concept to pin it on the right, then use quick toggle to swap sides.

Refine

Refinement is now a real image generation step. Pick the approved concept, choose one major attribute group to change, provide the new target value, and set the denoise strength used in the backend refinement workflow.

Select and approve a concept before running refinement.

Strength mapping: subtle = 0.25, medium = 0.45, strong = 0.65. The selected concept is used as the img2img parent, and locked attributes are restated in the refine prompt.

Layer Review

This stage is still placeholder. It remains useful for downstream smoke testing, but the UI now says plainly that these layers are synthetic stand-ins rather than extracted concept art.

Downstream warning: the current layer build still uses a synthetic part library.

Rig Review

The rig is still generated procedurally for the placeholder downstream path. Use this stage for deterministic renderer validation, not as evidence of concept-to-rig extraction maturity.

No rig data yet.

Production

The deterministic renderer remains in place. It is still useful for testing sheet packing, pivot consistency, and exports, but it sits on top of placeholder layers and rig data.

Production warning: renderer outputs are deterministic, but their upstream layer and rig sources are still synthetic.
