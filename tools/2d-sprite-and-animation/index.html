Setup

Start with a project name, then describe the character in plain language. The wizard uses these fields to build a stronger concept prompt tuned for readable Ashen Hollow sprites.

Start here. Give the sprite a project name first. You can add the full character description in the next step.

Describe the character like you would to an artist. Focus on role, mood, silhouette, clothing, and the one item they carry. The wizard will steer the prompt toward readable side-view game art.

Pixel Lab style & advanced

Pixel Lab parameters are saved with the brief. Debug vs ComfyUI is not selectable here—it stays on the value already stored for this project (new projects default to offline debug). Phase 8 will remove legacy ComfyUI paths.

Status row “Generation mode” shows whether this project uses debug_procedural (offline placeholders) or ComfyUI for any remaining legacy steps. To change it for an existing project, edit brief.json or use the API—Phase 7 keeps the server branch intact without a user-facing toggle.
